# 용어장

| 용어 | 뜻 |
| --- | --- |
| HTML | 웹페이지의 구조와 내용을 만드는 마크업 언어 |
| CSS | 색상, 여백, 배치, 반응형 등 화면 스타일을 정하는 언어 |
| JavaScript | 버튼 클릭, 데이터 처리, 화면 변경 같은 동작을 만드는 언어 |
| DOM | 브라우저가 HTML을 객체 구조로 표현한 것 |
| 이벤트 리스너 | 클릭, 입력 같은 사용자 행동을 감지해 함수를 실행하는 코드 |
| 정적 사이트 | 서버에서 매번 페이지를 만들지 않고 HTML, CSS, JS 파일을 그대로 제공하는 웹사이트 |
| 배포 | 로컬 파일을 인터넷에서 접속 가능한 상태로 올리는 작업 |
| Vercel | 정적 사이트와 웹앱을 배포하는 클라우드 플랫폼 |
| Git | 코드 변경 이력을 저장하고 비교하는 도구 |
| GitHub | Git 저장소를 온라인에 보관하고 공유하는 서비스 |
| 저장소 | 코드와 문서, 변경 이력을 담는 프로젝트 공간 |
| 커밋 | 변경 내용을 하나의 이력 단위로 저장한 것 |
| 브랜치 | 독립적으로 작업하기 위한 코드 흐름 |
| API | 다른 서비스나 코드와 정해진 방식으로 데이터를 주고받는 통로 |
| API 키 | 외부 API를 사용할 권한을 증명하는 비밀 값 |
| 환경 변수 | 코드 밖에서 주입하는 설정 값 |
| SEO | 검색 엔진이 페이지를 이해하고 노출하기 쉽게 만드는 작업 |
| sitemap.xml | 검색 엔진에 페이지 목록을 알려주는 파일 |
| robots.txt | 검색 엔진 크롤러에게 접근 규칙을 알려주는 파일 |
| manifest | 웹앱 이름, 아이콘, 테마 색상 등 설치형 앱 정보를 담는 파일 |
| MVP | 가장 작지만 실제로 작동하고 검증 가능한 첫 제품 |
| CTA | 사용자가 누르거나 실행해야 할 핵심 행동 버튼이나 문구 |
| 분석 도구 | 방문, 클릭, 전환 같은 사용자 행동을 측정하는 도구 |
| 서버리스 함수 | 서버를 직접 운영하지 않고 필요한 API 코드만 실행하는 방식 |
| PWA | 웹사이트를 앱처럼 설치하고 사용할 수 있게 만드는 웹앱 방식 |
